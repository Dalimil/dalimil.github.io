December 2015

Network Graph of my Facebook Friends

I thought it would be cool to visualize a graph of people who are my friends on Facebook. Nodes would represent people, with an edge between two nodes whenever those two people were friends.

First of all, I had to get the data set - I needed to get the 'friend list' of every user that I am friends with. Unfortunately, Facebook API does not support this. I've read that it did support it to some extent in the past (API v1.0) but due to privacy concerns this functionality has been removed (since API v2.0 - April 2014).

The only option left was to write a script that would log in to Facebook and crawl the network graph.

Sure, I have already done something similar before (see my "Facebook 2015+1" project, where I programmatically posted over two thousand Facebook posts), but this got much more complicated. The recursive nature of the search was not an issue at all (I was only interested in 'friends of my friends' anyway - otherwise the graph would become enormous). The problem was to get a 'friend list' of a single user.

Do you know how Facebook only shows you the first 20 friends when you visit somebody's profile? How you need to scroll down and wait until Facebook asynchronously pulls more friend profiles in the background?
Well, that was the real challenge to do in my code.

And when I was almost done, my little bot started receiving 'Security Check Required'.
I am not going to say how I got past that, but it was most amusing when I figured it out.
To make the graph a bit more fancy, I also pulled my friends' profile images and set them as nodes.

When I finally had the whole graph of my Facebook friends, I produced a DOT language file as output. I then used Graphviz to visualize the graph, but instead of using dot, the default graph layout mode (which works particularly well for hierarchical graphs), I used circo. Circo is a graph layout program (still part of the Graphviz package) for circular layouts. It identifies biconnected components and draws the nodes of each component on a separate circle. Edge crossings within a circle are minimized by placing as many edges on the circle's perimeter as possible.

Light blue dot = me; Red dots = people with private friend list; Purple dots = everyone else

I was blown away by the results! The graph very clearly showed groups of people that had strong connections among them (connection = 'be friends with'), such as people from my secondary school, people who are part of the same university society etc.
It was very interesting to find these patterns only by looking at the connections between my friends. In other words, the clusters of nodes could be very well mapped to real life social groups. Another interesting observation is that even if you keep your friend list private, you can be easily placed into a 'group' based on the connections that your friends reveal.

Finally, I experimented with another graph drawing tool - Gephi. It implements several force-directed layout algorithms, such as Fruchterman-Reingold, but I found the results less appealing (in terms of their layout) than those generated by Graphviz. See for yourself...

November 2015

JPMorgan Chase - 'Code for Good' 2015 - Hackathon

Code for Good was a weekend hackathon in Glasgow (7-8th Nov 2015) organised by JPMorgan Chase.
There were 12 teams with 5 people each (mostly university students). Three non-profit organizations â€” Pancreatic Cancer Scotland, Geeks Without Bounds and Urban Roots â€” presented challenges that they were facing. Four teams were assigned to each organization and their task was to develop innovative technology solution for the problem that the given organization was facing â€” in just 24 hours.


My team (consisting of me and 4 other Edinburgh students) developed a web application for Urban Roots. We made it possible for volunteers to sign up through the website, choose projects that they want to take part in, and see an embedded Google Map with all their current projects.

At the same time, we built an Android app that provided similar functionality to the website, but also allowed anyone to take a photo with their phone and thus report vandalism or any other issue seen in public places (Urban Roots made a request for this feature).


In addition, we created an admin section (on the website) where anyone from Urban Roots could easily manage existing projects, get in touch with volunteers, and also see a list of reported issues.

My team won the first stage, where each non-profit organization selected a team that came up with the best solution for their challenge. And after that, when each of the three winning teams presented their projects to a different set of judges (yes, we did the same presentation twice) we were selected as the best team overall and thus we became the winners of the entire hackathon. :-)

Facebook 2015+1

The aim of my Facebook 2015+1 project was to create 2015+1 Facebook posts. Each post shows a single number from 0 to 2015 (and the numbers are obviously in sorted order).
The solution to the task turned out to be quite tricky. Nevertheless, I accomplished it, with my program taking about an hour to produce those 2016 posts.


At first, it seemed that the most straightforward solution would be to use Facebookâ€™s Graph API Explorer or perhaps to obtain the Access Token via a custom-made Facebook application. However, when using the Facebook API you are limited to about 10 posts per day. Clearly, this wasnâ€™t the way to accomplish the task.

What I eventually did was to programmatically imitate a real user signing into Facebook and posting posts by sending HTTP requests in a similar way a web browser does. Still, there were some more issues such as Facebookâ€™s spam filters or also the posts not being in the correct order. These have all been dealt with. Tadaaa...
